## P1 Control Plane (server ↔ daemon) — Verified end-to-end

- [x] **Reconnect robustness + per-machine key rotation**: the server now rejects an unknown/removed `sk_machine_*` key with WS close `4001` (was a silent `ws.close()`), so the daemon stops the per-second reconnect storm — it backs off to the 30s cap and logs an actionable error; backoff resets only after the server accepts the connection, not on raw socket open. A new `POST /api/servers/:id/machines/:id/reconnect` rotates the key on the **same** machine row, and a **Reconnect** button on offline machines (`views/misc.tsx`) lets a lost-key machine rejoin without spawning a duplicate row. Verified: unit tests for the backoff/reject state machine (`connection.test.ts`, 3 cases), live daemon E2E (bogus/old key → `4001` + 30s backoff; rotated key → online on the same row, machine count unchanged; bootstrap happy-path regression), and browser (Reconnect button → new command modal)
- [x] **Reconnect catch-up + offline reachability (agents no longer go silently unreachable)**: when a machine reconnects, the server wakes only the agents on it that accumulated a *wakeable* backlog while offline (`reconnectCatchup.ts`) — they self-check the inbox and process every message missed offline, the symmetric counterpart to the human-side reconnect sync. The wake criterion mirrors `createMessage` (DM/@ always, ambient only with `inbox:receive`) so idle agents are never woken (no token-burn). Frontend: the Composer shows an offline reachability hint for the agents a message will reach (channel @-mention + DM peer — channels had none before), the misleading DM "queued and delivered" copy is corrected to match catch-up, and a lightweight global toast (`toast.tsx`) surfaces start/restart 503s and create-while-offline (`started:false`) that were silently swallowed. Verified: `isWakeable` unit tests (4 cases); live dev E2E (hard-offline @ → reconnect → catch-up `woke 1` → agent `message check` → reply; selective wake `scanned 2 woke 1`; ambient-no-scope agent not woken; online → no hint; start/create toasts; channel + DM offline banners). Soft-offline deliver / 30s cooldown / server-restart trigger are logic-verified (same `onReady` path), not separately E2E'd.
- [x] **Idle-sleep**: agent auto-kills process when idle (`OPEN_TAG_IDLE_MS`, default 10 min), resumes on next `--resume`/thread/resume; agent:stop/sleep linked

# Community-maintained continuation / 社区维护延续

> **Status — 2026-07-23.** This repository is now maintained as an independent community continuation of open-tag.
>
> **状态 — 2026-07-23。** 本仓库现作为 open-tag 的独立社区维护延续持续维护。

## English

The upstream recorded for this fork, [fancyboi999/open-tag](https://github.com/fancyboi999/open-tag), is currently unavailable. We do not know why and make no assertion about the original maintainer’s intent, status, or future plans.

[voyfei-lang/open-tag](https://github.com/voyfei-lang/open-tag) preserves the source history currently available in this repository. Please open new issues, proposals, and pull requests here. Report security vulnerabilities privately through [GitHub private vulnerability reporting](https://github.com/voyfei-lang/open-tag/security/advisories/new).

This is an independent, community-maintained repository. It does not claim affiliation with, authorization from, or endorsement by the original upstream maintainer. The Apache-2.0 license and all applicable copyright and attribution notices are retained; see [LICENSE](LICENSE) and [NOTICE](NOTICE).

The inherited upstream-sync workflow, old documentation deployment workflow, and old npm publishing workflow are paused while this project establishes maintainer-controlled release and documentation channels. The historical getopentag.com domains and @fancyboi999/open-tag-daemon package have not been transferred to this repository; do not treat them as current maintenance or release channels.

If you are an original contributor or maintainer and would like to coordinate, please open an issue here so the current maintainers can connect with you.

继承而来的上游同步、旧文档部署和旧 npm 发布工作流已暂停，等待建立由维护者控制的发布与文档渠道。历史上的 getopentag.com 域名和 @fancyboi999/open-tag-daemon 包均未移交给本仓库，请不要把它们视为当前维护或发布渠道。
